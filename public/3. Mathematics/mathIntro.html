Intro to Encryption

As any Fromsoft game or crash course history can show us, humanity lives in a constantly repeating cycle. From the social cycle theory to the average American workday, many aspects of modern society fall into this pattern. One of modern societies most important aspects, the internet, is currently stuck in perhaps the most prevalent cycle of all - the constant tug-of-war between freedom and safety.

Let’s take a step back. How is this a tug of war?

Well, you’ve probably heard of the social contract before. Put simply, it’s the idea that by placing some restraints on human behavior, it makes it possible for humanity to live in a community, which in turn moves society further both socially and morally. Within this community, our mental freedom to think differently than others has actually expanded, even if we are held subject to some higher power (government).

Encryption

What is it?

Encryption is the process of hiding data from potential attackers by changing the form of the data. This data has to be able to be decrypted, however, by either reversing the process of encryption or using a key to unlock the data. (more detail later)

How is it used?

Encryption has been used constantly throughout history. Originally, it was used mostly in the military, to prevent sensitive information from being intercepted. One of the most famous uses of encryption is the Caesar Cipher, a method Caesar used to protect his plans. It is a rather simple cipher - each letter of the alphabet is simply shifted left 3 spaces: D becomes A, G becomes A, and A wraps around to W, and so on. Ciphers continued to be used for the military,

Can I encryption my own messages?

If you’re going to send a physical letter, then a caesar cipher may protect your intrusive little brother from understanding anything, but beyond such a situation it really isn’t worth it to encrypt your own messages. Some of these methods are so complicated that using them to encrypt longer messages by hand can take days of straight work. If you want to encrypt digital messages, however, the process is a lot easier. If you have a little experience with coding, these methods can be very easy to automate, and can be done in a lot less time. It still isn’t worth it in many cases (as good messaging apps will encrypt for you), but if you need your messages to be completely secure, encrypting them yourself can be a good idea.

What are the most secure forms of encryption?

Currently, RSA is perhaps the most secure form (as it is the main asymmetric method), however it is slow and really only works one way. For a good two way method try AES - a bit more vulnerable to attacks (as both sides need the same key to unlock the message), but still secure. For absolute security, use RSA encryption to encrypt the AES key which was used to encrypt the original message.

Types of encryption

There are two main types of encryption - Symmetric and Asymmetric. Both have their advantages and disadvantages!

Symmetric Encryption

Symmetric encryption is the most simple form of encryption - and the one most people already know about. It comes from the idea of using a system to jumble text, so that only those who know the system can understand the original message. The key factor here is that the encryption goes two ways - this is why it is called symmetric - those who can encrypt the message using an algorithm can decrypt the message with the same algorithm.

Asymmetric Encryption

Asymmetric encryption is a bit harder to explain than symmetric. You see, when we think of an encrypted message, we usually think that one could only decrypt that message if you knew how it was encrypted in the first place. Well, Asymmetric encryption doesn’t exactly work like that. Asymmetric encryption works like email: anyone can encrypt a message and send it securely to you, but only you have the tools to decrypt the message. This means that the system used to asymmetrically encrypt a message is different from the system used to decrypt it.

Symmetric vs Asymmetric Encryption

Both systems work well, and usually can protect your message, but their are various strengths/weaknesses of both. Let's start with the first: security.

With security, asymmetric is the clear winner. As explained here (link to RSA encryption), Asymmetric can only be cracked using a brute-force attack, and with the right prime number length that can take hundreds of thousands of years to accomplish even on a supercomputer. Symmetric encryption, while still mostly safe, has been known to be cracked. Specifically DES encryption, which was the standard form a decade ago. Nowadays, with triple DES and AES (links to all these), symmetric encryption is much safer.

With Speed, however, symmetric is much faster. Symmetric just requires inputting a key into an algorithm and de-encrypting by using the same steps, just the opposite way. RSA, however, takes barely any time to encrypt, but a lot longer to decrypt. This is because decrypting RSA requires vast more amounts of calculations than DES and AES.

Because of these differences, Symmetric and Asymmetric are not used interchangeably. Each have their own important purposes - but the most common usage of both is in hybrid encryption, where RSA is used to encrypt a DES or AES key. This method is both fast (the RSA message is short), and very secure (the security of two encryptions).

What is the Diffie-Hellman key exchange?

Diffie-Hellman key exchange is a secure way of transmitting (and creating) passwords between two people. It is used to find AES and DES keys without any possible interception.

It works thusly:

Alice and Bob are smuggling gold across the canadian border. They know that Canadian mail customs will open and check every single package sent across, but will ignore any locks on said package.

But Alice and Bob don’t have a shared key for a lock - it would be too risky two both have a copy of the same key - instead, Alice and Bob have two separate locks, each with their own key.

Being cryptographic super geniuses, both of them independently come up with the smae plan to keep the gold safe.

Here’s how they do it.

Alice is in Canada with the gold. She buys an unmarked briefcase, locks it with her padlock, and sends it to Bob.

Bob has been waiting impatiently for a while now, and is worried for Alice. Seeing the briefcase on his doorstep with Alice’s lock on it, he smiles to himself, and nonchalantly brings it into his house. Unable to open the lock himself, he instead padlocks the case a second time with his personal lock, and sends it back to Alice.

Alice is also worried. Does Bob know what I’m trying to do? Have the pigs found us out?

But then Alice sees the briefcase, and a smile stretches across her face. Bob knew exactly what he was doing.

Alice unlocks her padlock, leaving the briefcase secure with only Bob’s lock, and then sends it back to Bob. This may work, she thinks. She needs money from selling this - it’s the only way her and her poor mother will be able to survive in this cold, calculating world.

And the briefcase gets through the border, and Bob unlocks it, selling all the gold, and then bitcoin-wiring some of the money back to Alice.

Lets go over that again.

A locks message M with A-lock, and sends it to B. B locks M with B-lock, and sends it back to A. A unlocks A-lock, and sends M to B. B unlocks B-lock, and can now read M.

What is a hash function?

Take any color of paint. This represents your message. Now, take an equal amount of neon green paint and mix it with your message. The resulting paint color is a mix of the two, but you could not separate them - you can’t un-mix paint. If you gave the mixture to someone else, they couldn’t tell you what your original color was. What you can do, however, is verify your color by mixing it with the green and giving it to that same person. Each original color has a different looking mixture with the green. In the same way, a hash function takes data and changes it into an unrecognizable and impossible to un-hash form with a very specific signature. That signature is incredibly important - if you hash the sentence Hello my name is Bob and Hello my nam is Bob, the hashes would come out completely different. One of the most common uses of this is in password storage. Web Servers never store your password in plaintext, they only hash your password and check to see if what you wrote matches the hash they copy down in your servers.

What's the difference between hashing and encryption?

What’s the difference between hashing and encryption? Both change data into a different form, but with encryption you can decrypt that data to change it back. With hashing, there’s no getting back what you hash, just as there’s no separating paint colors.

What is a digital signature?

To understand digital signatures, it may be best to understand RSA encryption. So go here (hyperlink to RSA encryption) and then come back.

Digital signatures are just RSA in reverse. In RSA, other people encrypt messages using your public key, and only you can decrypt them using your private key. With a digital signature, anyone can decrypt the message you send, but only you can encrypt it. This is not used for communication, as it would allow anyone with the public key to decrypt your messages: instead, the messages you send are proven to be yours, as only you have the tools available to encrypt.

What are some common attacks against encryption?

The most basic form of encryption attack is the brute force attack. This is basically just guessing someone's password by guessing every possible combination of letters - not the most efficient kind of attack, but easy to implement and code, and possible with super powerful computers breaking simple encryptions (single-DES).

Another kind of attack has to do with hashes: If a hacker can get into a website's server, that attacker may get access to the password hashes of that website’s user base. That hacker can then compare those password hashes with millions of the hashes of the most common passwords, and if any match, then that attack can now gain access to a user's account.

Fundamental mathematics of Encryption

What is modular Arithmetic?

Before understanding any of the math for RSA encryption (or any kind of encryption at all,) you have to first understand modular arithmetic.

Luckily (if you live in the United States), you’ve already done modular arithmetic before. Very often, in fact.

Now, this may not be true, but imagine the time is 4’oclock pm. The hour hand of a clock is on the 4, and the minute hand is on the 12.

Now think of the amount of time that has passed since the beginning of the day. If you did basic addition right, you’d find that it’s been 16 hours since the days technically began.

The clock, however, shows 4, not 16. This is because after 12 hours, the time resets to 0, and then goes back to 4.

Another way to represent this would be 16 mod 12.

You see, modular arithmetic just designates a number at which your count “resets”, so to say. So 4 mod 12 is still 4, yet 13 mod 12 is 1.

And it can reset multiple times: 30 mod 12 is 6, because 30 - 12 = 18, after 18 hours it would be 6 oclock.

And it doesn’t just have to reset after 12, either. You can mod other numbers:

For example, 5 mod 3 would equal 2: 1, 2, 3, (resets back to 0) 1, 2

10 mod 5 is 0, and 1 mod 2 is 1.

This kind of arithmetic is very important to all kinds of encryption (and many other kinds of computer code).

How does binary work?

Think of the powers of two, starting at 2^0. 1, 2, 4, 8, 16, et. Now, think of them as a series of switches on a wall - where a big calculator adds all the switches you’ve turned on - meaning that if you had the 0th switch (2^0) turned on, the calculator would read 1, and if you had the 0th (2^0) and 1st (2^1) switch turned on, the calculator would read (1+2=) 3.

Binary is just a physical representation of this, with a 1 meaning the switch is on, and a 0 meaning it’s off. (One caveat: the list of powers begins at the right and heads to the left)

Let’s break down a number:

101001 = 41

On our wall of flipped switches, the furthest switch (the left most number) would be the switch representing 32, and that is turned on (equals 1) so that’s the first number to be added. The next switch, 16, is not added, as the value representing it is 0, so it is turned off. The only other 1s represent the 8 and 1 switch, so those are added as well.

32 + 8 + 1 = 41.

A bitwise operation takes two long lines of bits and compares them, one pair at a time. For each pair, a single bit is outputted.

For example, a bitwise XOR (perhaps the most common bitwise operation in encryption), takes two bytes (these are just for example-it doesn’t have to be 8 bits long) and outputs an 8 bit result based on the following rules

If a 1 and a 1 is compared, it outputs 0
If 0 and a 1 are compared, it outputs 1
If a 1 and a 0 are compared, it outputs 1
If a 0 and a 0 are compared, it outputs 0

Bitwise & (and):

If a 1 and a 1 is compared, it outputs 1 If 0 and a 1 are compared, it outputs 0 If a 1 and a 0 are compared, it outputs 0 If a 0 and a 0 are compared, it outputs 0
Bitwise or:
If a 1 and a 1 is compared, it outputs 1 If 0 and a 1 are compared, it outputs 1 If a 1 and a 0 are compared, it outputs 1 If a 0 and a 0 are compared, it outputs 0
