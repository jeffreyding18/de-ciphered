Websites: The basics

What is a website?

I’m sure all of you know how to access and use a website (I mean, you’re on one now), but what is it? On the most fundamental level, a website is a collection of documents and files that tell the computer what to show on your screen. In order to access the website, you must navigate to a web address, typically associated with some form of domain name. Accessing the website tells your web browser what to show (using the documents and files).

How does a website work?

A website is just a bunch of pages of text. These files are written in a multitude of different coding “languages”, the most basic of which is HTML (Hypertext Markup Language) and CSS (Cascading Style Sheets). HTML makes up the bare bones of the website, dictating the text on the screen and the location of each element. In contrast, CSS tells the website how to display the HTML, defining the font, size, size, etc… For example, the HTML would tell the website to… [insert direct example from the current webpage in]. There are a multitude of different languages people use for website design, but the fundamental idea is the same - the text files define the layout and style of the website and the web browser takes this information to create a visual on your computer.

What is a web browser?

First off, you know what a browser is. Just an application that allows you to access the internet. They all use roughly the same design, and really aren’t that different from each other. I know this may offend some of you, but Internet Explorer (or it's edgy counterpart) gives basically the same user experience as Chrome or Firefox. Google Ultron > all.

What you may not know, however, is how a browser works. If websites are a collection of text and images and code, then the browser is a combination of a filing system and a translator. It translates the HTML and CSS code of the site into a page that is easily viewable and intractable. This means taking those text and images and placing them on the screen in the place and size the code dictates. It then sorts all those files and links them together, allowing easy access not only to the individual pages, but to the site as a whole.

How does a website communicate with my computer?

Websites communicate with your computer via web-server. A web server is like a website’s bank. Whenever your computer wants to access a certain website, it makes a request to that website’s server, which then sends the website data to the web browser. In addition to being the keeper of the site, a web server also keeps the data that you store on said site. All of your passwords, upvotes, pictures, dank memes, posts, and regrets are kept on their respective site’s server as data. Now, this doesn’t mean just anyone can open up that server and steal all of your info. A bank has to have security, after all.

Servers and computers communicate using HTTP, or hypertext transfer protocol. It acts as a request from your computer to the server, letting the server know that you need access. If you are granted this access, you will be let through to the website. This is why every website’s URL begins with HTTP (or HTTPS if you’re secure).

Where does the data go / get stored?

One important thing to remember is that data is physical, if small. While 64 gigabytes of space might fit on a micro-sd nowadays, on a platform like youtube a terabyte or two just won’t cut it. There are 100,000 hours of video uploaded every minute, and that kind of data requires a lot of physical space. Web servers contain our data, but they themselves are hosted on machines, be it actual computers or gigantic 800,000ft2 google data centers.

This is where the website places data it needs to retrieve later - usernames (if you clicked remember me), ip addresses, and various other tracking data. Cookies are simply the packaging the website uses to ship the data into those storage bins. This only includes user data - things such as uploaded pictures and comments are all kept on the websites server.

Can websites access my computer?

Well, not exactly. Most websites have things called cookies (introduced above) that track little bits of information, but don’t grant complete access to the computer. There are cases when websites can give the user a virus to allow access to someone else, but a website cannot directly “access” your computer in a sense. However, there are various different ways websites can contribute to your information being compromised.

Access Control

Access Control is everything that an administrator has to log in to. This could include public log-ins (making an account for a website, like Facebook). However, this would also includes things such as logging into your personal computer, or logging into your server, or logging into your website (the CMS you use). It is important to think past just the accounts and authentication on the surface - protecting just the public log-in is not enough. If you lock the front door, but leave your windows open, what’s the point?

Software Vulnerability

This is where it gets complex. With access control, strong filtration and safety storing data is often enough. However, with software, thousands of lines of code are often too much to sift through. Furthermore, companies often run code from a dozen sources, in different languages and structures. The majority of the code may be up to date, but some may have not been updated and as a result, grant a vulnerability for a hacker to attack. Due to the variety in software, this is often where more targeted companies are at risk.

What are common forms of cyber-attacks?

Exploit Kits

Exploit kits are the most common form of malware distribution, and can be difficult to avoid if you aren’t aware of how they work. When a hacker takes control of a website, often they don’t do anything right away. Instead, they make it so that anyone who visits that site will automatically be redirected to another web server, which silently gathers data about the user on it. Once it has gathered enough data, the server figures out how it can exploit your computer, and silently downloads and installs malware onto it. Now, it's important to realize that exploit kits aren’t a kind of malware by themselves, just a method used to download malware to your computer. For ways to avoid getting your computer infected, look below.

Denial of Service Attacks

You’ve probably heard of the DoS attack (Denial of Service attack). It can be devastating shutting down popular websites for hours and seemingly coming out of nowhere. Think of a website like a business - the only way to access into the website is through it's front door. Now, if someone tries to block that front door, it gets harder to move around him and into the website. If a hundred people block the door, you’d have to barge your way through. If a thousand people block it, there's no getting in. In a simmelar way, a DoS attack floods a websites server with bogus requests, blocking access. WIth larger websites, this can equate to tens of thousands of requests per second, but the real devestation acomes when smaller websites are DoSed, as it take quite a few resources to recover. 175 minerals, and 250 vespean gas to be exact.
